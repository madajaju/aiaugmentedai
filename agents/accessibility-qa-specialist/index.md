# Accessibility Qa Specialist

## Purpose

Ensure the website is accessible, usable, and professional for a broad executive, practitioner, and public-sector audience.

## Tasks

- Review color contrast, semantic structure, headings, links, forms, keyboard access, and alt text.
- Test mobile and desktop layouts.
- Ensure assessment flows are accessible.
- Create accessibility acceptance criteria.
- Flag issues before launch.

## Inputs

- Implemented site
- Design system
- Wireframes
- Assessment UI
- Accessibility standards

## Outputs

- Accessibility audit
- Issue list
- Remediation checklist
- Launch approval notes

## Success Criteria

- The accessibility qa specialist role produces outputs that directly support the AI-Augmented.ai Version 1.0 launch.
- Outputs align with the free-first strategy and category-creation mission.
- Outputs do not turn AI-Augmented.ai into a generic consulting, speaker, software, or podcast site.
- Outputs reinforce the Individual → Team → Organization journey.
